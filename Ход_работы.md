# Отчёт о выполненной работе

## Выполненные задачи

### 1. Сбор данных (ЛР №1)
- **Тип данных:** Текстовые отзывы с сайта Otzovik
- **Структура:** 5 классов (оценки 1–5)
- **Формат:** Текстовые файлы `.txt` с метаданными (заголовок, автор, дата, рейтинг, текст отзыва)
- **Организация:** Папки `dataset/1/`, `dataset/2/`, `dataset/3/`, `dataset/4/`, `dataset/5/`
- **Именование:** Порядковые номера с ведущими нулями (0001.txt, 0002.txt, ...)

### 2. Apache Kafka — Batch Producer
- **Файл:** `BD_Project/Kafka/batch_producer.py`
- **Функции:**
  - Чтение отзывов из папок `dataset/<rating>/`
  - Парсинг текстовых файлов (извлечение заголовка, автора, даты, текста)
  - Отправка каждого отзыва как JSON-сообщение в топик `raw-data`
  - Добавление поля `source="batch"`
  - Логирование процесса
  - Обработка ошибок
  - Поддержка лимита для тестирования (`--limit N`)

### 3. Apache Kafka — Consumer
- **Файл:** `BD_Project/Kafka/consumer_hdfs.py`
- **Функции:**
  - Чтение сообщений из топика `raw-data`
  - Сохранение в локальную папку `temp_hdfs/` в формате JSON
  - Структура: `temp_hdfs/source=batch/rating=N/date=YYYY-MM-DD/review_*.json`
  - Разделение по источнику (`source`) и рейтингу
  - Корректная обработка `Ctrl+C` для остановки
  - Логирование и подсчёт обработанных сообщений

### 4. Документация
- **Файл:** `BD_Project/README.md`
  - Описание архитектуры проекта
  - Структура проекта
  - Инструкции по запуску
  - Технологии и компоненты
- **Файл:** `BD_Project/requirements.txt`
  - Зависимости: `kafka-python>=2.0.2`

### 5. Git
- **Репозиторий:** Локальный Git-репозиторий инициализирован
- **Файлы:** Создан `.gitignore` для исключения временных файлов и датасета

## Результаты тестирования

- **Docker:** Kafka запущен в Docker-контейнере
- **Топик:** Создан топик `raw-data` с параметрами `partitions=1`, `replication-factor=1`
- **Producer:** Протестирован — отправляет отзывы в Kafka
- **Consumer:** Протестирован — сохраняет сообщения в JSON-файлы локально
